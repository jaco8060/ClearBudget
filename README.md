# Modern Budget Tracker 💸

A robust, cloud-synced personal finance application built with React, Vite, Material UI, and powered seamlessly by Cloudflare's serverless edge ecosystem. 

This application abandoned traditional `localStorage` constraints in favor of a globally persistent Cloudflare D1 SQL Database, allowing users to safely log in and sync their financial data seamlessly across their desktop and mobile devices.

## ✨ Features

- **Robust Authentication:** Secure cookie-based login and registration system handled entirely via Cloudflare Pages middleware and SQLite sessions.
- **Real-Time Data Visualizations:** An interactive, animated pie chart built using `recharts` breaks down your weekly recurring vs. extra spends compared to your remaining budget.
- **Zero-Based Budgeting Tools:** Track comprehensive financial health across 3 major categories: Earnings, Recurring Monthly Expenses, and spontaneous Extra Spends.
- **Edge Cloud Backend:** Fully serverless architecture utilizing Cloudflare Pages Functions (`functions/api/`) connecting directly to a Cloudflare D1 database for lightning-fast latency anywhere in the world.
- **Sleek Dark Mode UI:** A gorgeous, responsive, and mobile-first interface designed with `@mui/material`.

## 🛠️ Technology Stack

- **Frontend:** React 19, Vite, Material UI, Recharts, Emotion
- **Backend/API:** Cloudflare Pages Functions (Serverless node environment)
- **Database:** Cloudflare D1 (Serverless edge SQLite database)
- **Tooling:** Wrangler CLI

## 🚀 Getting Started Locally

To run this application on your own machine, you must leverage Cloudflare's `wrangler` CLI to emulate the database environment.

### 1. Prerequisites
Ensure you have `npm` and the `wrangler` CLI installed.
```bash
npm install -g wrangler
npm install
```

### 2. Configure the Database
Authenticate with your Cloudflare account and create a D1 Database:
```bash
wrangler login
wrangler d1 create budget-db
```
Take the `database_id` generated by that command, open `wrangler.toml`, and update the `database_id` configuration line.

### 3. Initialize the SQL Schema
Seed your fresh database with the necessary tables (`users`, `sessions`, `earnings`, `monthly_expenses`, `extra_spends`):
```bash
# Set up your local development database
npm run db:init:local
```

### 4. Run the Dev Server
Instead of a standard `vite` command, we use `wrangler` to run a proxy server that connects Vite to our simulated Cloudflare backend:
```bash
npm run pages:dev
```
Your app will be automatically served at `http://localhost:8788`.

## 🌎 Deploying to Production

When you are ready to publish the app to the internet, we have included a single deploy command:

```bash
npm run deploy
```

**⚠️ CRITICAL PRODUCTION STEP:** 
Cloudflare Pages requires you to securely authorize the Database connection for live environments:
1. Go to your **Cloudflare Dashboard** -> **Workers & Pages**.
2. Select your `budget-app` project.
3. Go to **Settings** -> **Functions** -> **D1 database bindings**.
4. Click **Add binding**. Set the variable name exactly as **`DB`**, and assign it to your `budget-db`.
5. Run `npm run deploy` one more time!
